## Overview

The Camera Control Interface enables precise specification of camera trajectories for video generation in WorldStereo. This interface allows you to define camera poses, movements, and parameters that guide the video generation process while maintaining 3D consistency.

## Camera Trajectory Specification

### CameraTrajectory

Define a camera trajectory for video generation.

<ParamField path="poses" type="List[CameraPose]" required>
  A sequence of camera poses defining the trajectory. Each pose specifies the camera position and orientation at a specific frame.
</ParamField>

<ParamField path="num_frames" type="int" required>
  Total number of frames to generate along the trajectory.
</ParamField>

<ParamField path="interpolation" type="str" default="linear">
  Interpolation method between poses. Options:
  - `linear`: Linear interpolation
  - `spline`: Smooth spline interpolation
  - `bezier`: Bezier curve interpolation
</ParamField>

<ParamField path="fps" type="int" default="24">
  Frames per second for the generated video.
</ParamField>

### CameraPose

Represents a single camera pose in 3D space.

<ParamField path="position" type="List[float]" required>
  Camera position in 3D space as `[x, y, z]` coordinates.
</ParamField>

<ParamField path="rotation" type="Union[List[float], Dict]" required>
  Camera orientation. Can be specified as:
  - Quaternion: `[w, x, y, z]`
  - Euler angles: `{"roll": float, "pitch": float, "yaw": float}` (in degrees)
  - Rotation matrix: 3x3 nested list
</ParamField>

<ParamField path="frame_index" type="int" required>
  Frame index where this pose should be applied.
</ParamField>

<ParamField path="fov" type="float" default="60.0">
  Field of view in degrees.
</ParamField>

## Camera Intrinsics

### CameraIntrinsics

Define camera intrinsic parameters.

<ParamField path="focal_length" type="float" required>
  Focal length in pixels.
</ParamField>

<ParamField path="principal_point" type="List[float]" default="[width/2, height/2]">
  Principal point coordinates as `[cx, cy]`.
</ParamField>

<ParamField path="width" type="int" required>
  Image width in pixels.
</ParamField>

<ParamField path="height" type="int" required>
  Image height in pixels.
</ParamField>

<ParamField path="distortion" type="List[float]" default="[0, 0, 0, 0, 0]">
  Lens distortion coefficients `[k1, k2, p1, p2, k3]`.
</ParamField>

## Usage Examples

### Basic Camera Trajectory

<CodeGroup>

```python Python
from worldstereo.camera import CameraTrajectory, CameraPose

# Define keyframe poses
poses = [
    CameraPose(
        position=[0, 0, 0],
        rotation={"roll": 0, "pitch": 0, "yaw": 0},
        frame_index=0
    ),
    CameraPose(
        position=[1, 0, 0],
        rotation={"roll": 0, "pitch": 0, "yaw": 0},
        frame_index=30
    ),
    CameraPose(
        position=[2, 0.5, 0],
        rotation={"roll": 0, "pitch": 10, "yaw": 0},
        frame_index=60
    )
]

# Create trajectory
trajectory = CameraTrajectory(
    poses=poses,
    num_frames=60,
    interpolation="spline",
    fps=24
)
```

### Circular Camera Motion

```python
import numpy as np
from worldstereo.camera import CameraTrajectory, CameraPose

# Generate circular trajectory
num_frames = 120
radius = 2.0
height = 1.0

poses = []
for i in range(0, num_frames, 10):
    angle = 2 * np.pi * i / num_frames
    x = radius * np.cos(angle)
    z = radius * np.sin(angle)
    yaw = np.degrees(angle + np.pi/2)
    
    poses.append(CameraPose(
        position=[x, height, z],
        rotation={"roll": 0, "pitch": 0, "yaw": yaw},
        frame_index=i
    ))

trajectory = CameraTrajectory(
    poses=poses,
    num_frames=num_frames,
    interpolation="spline"
)
```

## Trajectory Utilities

### load_trajectory()

Load camera trajectory from file.

<ParamField path="path" type="str" required>
  Path to trajectory file. Supported formats:
  - `.json`: JSON trajectory specification
  - `.txt`: COLMAP-style camera poses
  - `.xml`: OpenCV camera calibration format
</ParamField>

<ParamField path="format" type="str" default="auto">
  Format specification. Auto-detected from file extension if not provided.
</ParamField>

**Returns:**

<ResponseField name="trajectory" type="CameraTrajectory">
  Loaded camera trajectory object.
</ResponseField>

### save_trajectory()

Save camera trajectory to file.

<ParamField path="trajectory" type="CameraTrajectory" required>
  Camera trajectory to save.
</ParamField>

<ParamField path="path" type="str" required>
  Output file path.
</ParamField>

<ParamField path="format" type="str" default="json">
  Output format (`json`, `colmap`, or `opencv`).
</ParamField>

### interpolate_poses()

Interpolate between camera poses.

<ParamField path="start_pose" type="CameraPose" required>
  Starting camera pose.
</ParamField>

<ParamField path="end_pose" type="CameraPose" required>
  Ending camera pose.
</ParamField>

<ParamField path="num_steps" type="int" required>
  Number of interpolation steps.
</ParamField>

<ParamField path="method" type="str" default="slerp">
  Interpolation method for rotations:
  - `slerp`: Spherical linear interpolation (recommended)
  - `linear`: Linear interpolation
</ParamField>

**Returns:**

<ResponseField name="poses" type="List[CameraPose]">
  List of interpolated camera poses.
</ResponseField>

## Example Usage

```python
from worldstereo.camera import load_trajectory, save_trajectory

# Load trajectory from COLMAP format
trajectory = load_trajectory(
    "camera_poses.txt",
    format="colmap"
)

# Save as JSON
save_trajectory(
    trajectory,
    "trajectory.json",
    format="json"
)
```

## Coordinate System

<Note>
  WorldStereo uses a right-handed coordinate system:
  - **X-axis**: Right
  - **Y-axis**: Up
  - **Z-axis**: Backward (into the scene)
  
  Camera looks along the negative Z-axis by default.
</Note>

## Best Practices

<Info>
  **Smooth trajectories**: Use spline interpolation with sufficient keyframes for smooth camera motion.
  
  **Consistent scale**: Maintain consistent position scales across poses to ensure proper geometric memory integration.
  
  **Reasonable motion**: Avoid extreme camera velocities between frames to maintain video quality and 3D consistency.
  
  **Frame spacing**: Space keyframes 10-30 frames apart for optimal interpolation quality.
</Info>
